The chief problem with solving this problem is organizing the search process so that we do not repeatedly solve the same subproblems. We will do this organization through the use of the kk-path. Define a k-path from vertex vv to vertex uu to be any path whose intermediate vertices (aside from vv and uu) all have indices less than kk. A 0-path is defined to be a direct edge from vv to uu. Define Dk(v,u)\mathrm{D}_k(v, u) to be the length of the shortest kk-path from vertex vv to vertex uu. Assume that we already know the shortest kk-path from vv to uu. The shortest (k+1)(k+1)-path either goes through vertex kk or it does not. If it does go through kk, then the best path is the best kk-path from vv to kk followed by the best kk-path from kk to uu. Otherwise, we should keep the best kk-path seen before. Floydâ€™s algorithm simply checks all of the possibilities in a triple loop. Here is the implementation for Floydâ€™s algorithm.
